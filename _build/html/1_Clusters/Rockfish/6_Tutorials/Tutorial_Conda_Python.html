Conda PythonÔÉÅ

This tutorial guides you through creating your own Conda environment with Python on Rockfish.

Connect to Rockfish (Login Node)ÔÉÅ

Open a terminal and run:

ssh YourUserID@login.rockfish.jhu.edu

Enter your Rockfish password when prompted.

Warning

Since computational tasks shouldn‚Äôt run on login nodes, you‚Äôll next connect to a compute node.

Connect to a Compute NodeÔÉÅ

interact -p shared -n 6 -t 02:00:00

Parameter GuideÔÉÅ

  • -p: Partition (express, shared, parallel)

  • -n: Number of cores - express ‚Üí up to 4 cores - shared ‚Üí up to 32 cores - parallel ‚Üí up to 48 cores

  • -t: Job time in HH:MM:SS - express: up to 8 hours - shared: up to 36 hours - parallel: up to 72 hours

Load Anaconda ModuleÔÉÅ

module reset
module load anaconda3/2024.02-1

Create Conda EnvironmentÔÉÅ

conda create --name mypy_env python=3.11 -y

ExplanationÔÉÅ

  • --name mypy_env: Name of your environment

  • python=3.11: Python version

  • -y: Auto-confirm installation

Set Conda ChannelsÔÉÅ

conda config --env --add channels defaults
conda config --env --add channels bioconda
conda config --env --add channels conda-forge
conda config --env --set channel_priority strict

Activate the EnvironmentÔÉÅ

conda activate mypy_env

Once activated, all installed packages will live inside this environment.

Install PackagesÔÉÅ

Example: Install matplotlib

# Using pip
pip install matplotlib

# Or using conda
conda install -c conda-forge matplotlib -y

List installed packages:

conda list

Deactivate Environment (When Done)ÔÉÅ

conda deactivate

To reactivate later:

module load anaconda3/2024.02-1
conda activate mypy_env

List all your environments:

conda env list

Exit the Compute NodeÔÉÅ

exit
